# 数据库与缓存 — AI 应用工程师必备

## 1. 向量数据库

### 为什么重要
RAG（检索增强生成）是 AI 应用最常见架构，核心就是向量检索。面试必问。

### 核心概念

```
原始文本 → Embedding 模型 → 向量 [0.1, -0.3, 0.8, ...] → 存入向量数据库
                                                              ↓
用户提问 → Embedding 模型 → 查询向量 → 向量数据库相似度搜索 → 返回 Top-K 相关文档
```

### 相似度度量

| 方法 | 公式/说明 | 适用场景 |
|------|----------|---------|
| **余弦相似度** | `cos(θ) = A·B / (|A|×|B|)` | 最常用，关注方向而非长度 |
| **欧几里得距离** | `√(Σ(Ai - Bi)²)` | 关注绝对距离，需要归一化 |
| **点积** | `A·B = Σ(Ai × Bi)` | 向量已归一化时等价余弦相似度 |

### PostgreSQL + pgvector（最推荐）

```sql
-- 启用扩展
CREATE EXTENSION vector;

-- 创建表，embedding 用 vector 类型
CREATE TABLE documents (
    id SERIAL PRIMARY KEY,
    content TEXT NOT NULL,
    source TEXT,
    embedding VECTOR(1536)  -- OpenAI text-embedding-ada-002 是 1536 维
);

-- 创建 IVFFlat 索引（近似搜索，速度最快）
CREATE INDEX ON documents
USING ivfflat (embedding vector_cosine_ops)
WITH (lists = 100);

-- 相似度搜索
SELECT content, source, 1 - (embedding <=> query_embedding) AS similarity
FROM documents
ORDER BY embedding <=> query_embedding  -- <=> 是余弦距离
LIMIT 5;
```

### pgvector 常用操作符
```sql
embedding <=> query   -- 余弦距离（越小越相似）
embedding <-> query   -- 欧几里得距离
embedding <#> query   -- 负内积（越大越相似）
```

### Python 操作示例
```python
import asyncpg
import numpy as np
from openai import AsyncOpenAI

class VectorStore:
    def __init__(self, dsn: str, openai_key: str):
        self.dsn = dsn
        self.client = AsyncOpenAI(api_key=openai_key)

    async def embed(self, text: str) -> list[float]:
        resp = await self.client.embeddings.create(
            model="text-embedding-3-small",
            input=text
        )
        return resp.data[0].embedding

    async def add_document(self, content: str, source: str):
        embedding = await self.embed(content)
        conn = await asyncpg.connect(self.dsn)
        try:
            await conn.execute(
                "INSERT INTO documents (content, source, embedding) VALUES ($1, $2, $3)",
                content, source, embedding
            )
        finally:
            await conn.close()

    async def search(self, query: str, top_k: int = 5) -> list[dict]:
        query_emb = await self.embed(query)
        conn = await asyncpg.connect(self.dsn)
        try:
            rows = await conn.fetch("""
                SELECT content, source,
                       1 - (embedding <=> $1::vector) AS similarity
                FROM documents
                ORDER BY embedding <=> $1::vector
                LIMIT $2
            """, query_emb, top_k)
            return [{"content": r["content"], "source": r["source"],
                     "similarity": r["similarity"]} for r in rows]
        finally:
            await conn.close()
```

### 其他向量数据库对比

| 数据库 | 特点 | 适用 |
|--------|------|------|
| **Chroma** | 轻量、Python 原生、开发友好 | 原型、小项目 |
| **Milvus** | 分布式、十亿级向量、GPU 加速 | 大规模生产 |
| **Pinecone** | 全托管、零运维 | 不想自己管基础设施 |
| **Weaviate** | GraphQL 接口、混合搜索 | 需要结构化+向量混合 |
| **Qdrant** | Rust 实现、高性能 | 对延迟敏感 |

## 2. Redis

### AI 应用中的 Redis 用法

#### 1) 语义缓存 — 减少 LLM 调用成本
```python
import redis.asyncio as redis
import hashlib
import json

class LLMCache:
    def __init__(self, redis_url: str, ttl: int = 3600):
        self.redis = redis.from_url(redis_url)
        self.ttl = ttl  # 缓存 1 小时

    def _hash(self, prompt: str) -> str:
        return hashlib.sha256(prompt.encode()).hexdigest()

    async def get(self, prompt: str) -> str | None:
        key = f"llm_cache:{self._hash(prompt)}"
        cached = await self.redis.get(key)
        return cached.decode() if cached else None

    async def set(self, prompt: str, response: str):
        key = f"llm_cache:{self._hash(prompt)}"
        await self.redis.setex(key, self.ttl, response)

# 语义缓存（扩展版） — 相似问题命中同一缓存
class SemanticCache(LLMCache):
    async def get_similar(self, prompt: str, threshold: float = 0.95) -> str | None:
        """用向量相似度匹配缓存"""
        emb = await embed_text(prompt)
        # 用 RedisSearch 的向量搜索能力
        results = await self.redis.ft("idx:cache").search(
            query=f"@embedding:[VECTOR_RANGE {threshold} $vec]=>{{$vec: {emb}}}",
            query_params={"vec": np.array(emb, dtype=np.float32).tobytes()}
        )
        return results.docs[0].response if results.docs else None
```

#### 2) 限流 (Rate Limiter)
```python
class RateLimiter:
    def __init__(self, redis_client, max_requests: int, window: int):
        self.redis = redis_client
        self.max_requests = max_requests
        self.window = window  # 秒

    async def is_allowed(self, user_id: str) -> bool:
        key = f"rate_limit:{user_id}"
        current = await self.redis.incr(key)
        if current == 1:
            await self.redis.expire(key, self.window)
        return current <= self.max_requests
```

#### 3) 会话管理
```python
class ChatSession:
    async def save(self, session_id: str, messages: list[dict], ttl: int = 1800):
        """保存对话历史，1800 秒过期"""
        await self.redis.setex(
            f"session:{session_id}",
            ttl,
            json.dumps(messages)
        )

    async def load(self, session_id: str) -> list[dict]:
        data = await self.redis.get(f"session:{session_id}")
        return json.loads(data) if data else []

    async def append(self, session_id: str, message: dict):
        messages = await self.load(session_id)
        messages.append(message)
        await self.save(session_id, messages)
```

## 3. PostgreSQL 高级用法

### pgvector + 传统查询混合
```sql
-- 同时按向量相似度和结构化条件过滤
SELECT content, source,
       1 - (embedding <=> $1::vector) AS similarity
FROM documents
WHERE source = 'handbook'                        -- 结构化条件
  AND created_at > NOW() - INTERVAL '30 days'    -- 时间范围
  AND 1 - (embedding <=> $1::vector) > 0.7       -- 相似度阈值
ORDER BY embedding <=> $1::vector
LIMIT 10;
```

### JSONB — 灵活元数据存储
```sql
-- 存储文档的结构化信息
CREATE TABLE knowledge_base (
    id SERIAL PRIMARY KEY,
    content TEXT,
    metadata JSONB,              -- 灵活的元数据
    embedding VECTOR(1536)
);

-- JSONB 查询
SELECT * FROM knowledge_base
WHERE metadata @> '{"category": "技术文档"}'        -- 包含
  AND metadata ->> 'author' = '张三';              -- 等于

-- GIN 索引加速 JSONB 查询
CREATE INDEX ON knowledge_base USING GIN (metadata);
```
